JPEG

 

The upper half of the dialog lets you set a quality level for JPG images. Move the slider control to the desired setting. Lower settings take less space, but deliver less quality. Higher settings are better, but take more file space. The default setting is 80, which is fine for most images.

 

Check the grayscale option to save the image as that. Please remember if you activate this option!

Click the Save as progressive JPG box (like many images from internet) to activate that option. Click either again to remove the choice.

 

Click the Disable chroma color subsampling box if you want to achieve best color quality during JPG compression, which means 1x1 pixel blocks and larger JPG file. Normal block size is 2x2 pixels (enabled color subsampling).

 

You can also control the saving of original EXIF/IPTC/XMP/Comment informations into the new JPG file.

These options (IPTC/XMP/Comment) are valid if you convert/save a JPG to another JPG, other formats (like TIF or RAW formats with EXIF/IPTC data) are not yet supported.

The EXIF saving works from JPG to JPG but it can also work with many RAW/TIF files, if the EXIF data is saved at beginning of the file.

 

If you activate the option Try to save with original quality, IrfanView will try to estimate the quality and subsampling of the original JPG file and use these values for saving. If the original values can't be estimated/recovered, the current dialog settings for quality/subsampling are used.

 

You can set the approx. kilobyte size of the result JPG file. IMPORTANT NOTE: a JPG file/image can't be saved smaller than a JPG file/image with quality of 10, means there are certain JPG limitations regarding minimal file size. If you set a smaller file size than JPG (with quality of 10, the minimum) can compress, you may get an error message or a JPG file saved with quality of 10.

 

Show preview dialog: after you click on the Save button, a preview dialog will be shown where you can adjust (again) the JPG save quality and/or file size.

 

 

GIF

 

The bottom half of the dialog has options for GIF images.

 

Click the Save interlaced box (like many images from internet) for that option.

 

Info:If the image contains transparency color (if the image has max. 256 colors, 8 BPP), it will be automatically used for saving, no need to set a custom transparency color.

 

Click Save transparent color to set a Custom transparency for the result GIF image.

Choose transparent color during saving: later, you can click into a preview image to set the transparent color, this is the easy method.

Set transparency value to palette entry: type a number in the box below to use that palette entry as the transparent color. For more on this, see Edit Palette. If you have a true color image, you must first decrease the color depth to 256 colors to see the palette.

If you activate the option Use main window color for transparency, the current window color (Properties->Viewing) will be automatically set as transparent color.

 

Note: IrfanView can't save animated GIFs.

 

 

You can save the current/visible dialog values as a profile and later use/load the saved values.

 

 

After setting the options you choose, click Save to save the image and store the settings. To exit without saving the file or changed options, click Cancel.

 

The new settings will stay in effect until you change them again.
